# API Traceability Specification

## Purpose

Ensure every API request can be correlated end-to-end through a unique `X-Trace-Id` header generated by the frontend and propagated to all backend calls and responses.

## Requirements

### Requirement: Trace Id Generation

The system MUST generate a unique `X-Trace-Id` for every inbound request and every outgoing API call when one is not already present.

#### Scenario: New user request

- GIVEN a request enters the frontend with no `X-Trace-Id` header
- WHEN the request is processed
- THEN a unique `X-Trace-Id` is generated and associated with the request

#### Scenario: Existing trace id preserved

- GIVEN a request enters the frontend with an existing `X-Trace-Id` header
- WHEN the request is processed
- THEN the existing `X-Trace-Id` is preserved and reused

### Requirement: Trace Id on Outgoing API Calls

Every API fetch call from `src/lib/api.ts` MUST include the `X-Trace-Id` header. The same trace id MUST be used for all calls triggered by the same user action.

#### Scenario: Catalog fetch

- GIVEN the catalog page requests products
- WHEN `getProducts()` executes
- THEN the outgoing request contains an `X-Trace-Id` header

#### Scenario: Multi-step checkout

- GIVEN a checkout action triggers payment-intent and order creation calls
- WHEN the calls execute
- THEN both calls include the same `X-Trace-Id` header

### Requirement: Trace Id in Route Handlers

All Next.js API route handlers MUST include the `X-Trace-Id` header in outbound calls to Strapi, Stripe, or other backends. Route handlers SHOULD return the `X-Trace-Id` in response headers for client correlation.

#### Scenario: Payment intent route

- GIVEN `/api/create-payment-intent` receives a request with `X-Trace-Id`
- WHEN the route calls Stripe
- THEN the Stripe request includes the same `X-Trace-Id`
- AND the route response includes the `X-Trace-Id`

#### Scenario: Orders route

- GIVEN `/api/orders` receives a request with `X-Trace-Id`
- WHEN the route calls Strapi
- THEN the Strapi request includes the same `X-Trace-Id`

### Requirement: Trace Id on Errors

Error responses from API route handlers MUST include the associated `X-Trace-Id` so users and support can correlate failed requests.

#### Scenario: Failed backend call

- GIVEN an API route handler encounters a backend error
- WHEN it returns an error response
- THEN the response contains the `X-Trace-Id` header
